# jarvis 12 条铁律

1. 所有重要任务统一进入 `jarvis`，不绕开中枢。

2. 每个任务只能有一个主责 Agent，其他 Agent 只能协作，不能争夺控制权。

3. 研究、执行、归档必须分离，不能由同一个 Agent 静默包办。

4. 任何值得长期保留的结果，最终都必须进入知识层，由 `Knowledge Agent` 或知识流程统一收口。

5. 长期资产必须落地为文件，不能只存在聊天记录、隐藏记忆、向量索引或厂商私有存储中。

6. 文件组织按领域和资料类型划分，不按临时 Agent 实例划分。

7. 所有长期文档都必须带稳定 metadata，至少包含：`id`、`title`、`created_at`、`updated_at`、`domain`、`type`、`status`、`tags`、`visibility`。

8. 索引、缓存、向量库都必须可重建，原始文件才是真相源。

9. Agent 负责判断，Tool 负责执行，二者不能混为一体。

10. 发布、删除、外发、交易、关键配置修改都属于高风险动作，必须显式确认或受明确权限规则约束。

11. 对外沟通默认不能由研究型 Agent 直接发起，必须由执行型 Agent 或关系型 Agent 作为最后一跳。

12. 无论模型、框架、界面如何变化，`jarvis` 的规则、工作区和长期知识都必须保持可迁移、可理解、可延续。
